<p>As part of preparing <a href="http://marbl.library.emory.edu/findingaids/content.php?id=rushdie1000_1071">Salman Rushdie&rsquo;s papers</a> for presentation to researchers, some of the digital documents need to redacted. <a href="https://techknowhow.library.emory.edu/2009-marbl-redaction-recommendation">Evaluating redaction software feature lists</a> isn&rsquo;t enough, though: We also need to make sure the documents are in a format the redaction software understands.</p>
<p><a href="http://www.rapidredact.com/features.asp">Some redaction software is pretty flexible</a> in what it&rsquo;ll open. <a href="http://www.appligent.com/redax">Other programs</a> require you to convert the file to PDF, though that&rsquo;s typically <a href="https://createpdf.adobe.com/">not</a> <a href="http://www.gnu.org/software/a2ps/">very</a> <a href="http://www.google.com/search?q=convert+pdf">hard</a> either. Unless, of course, you&rsquo;re converting and redacting files from a twelve-year-old <a href="http://en.wikipedia.org/wiki/System_7_(Macintosh)">System 7</a> Mac, where some of those are backups from even older systems. There aren&rsquo;t a ton of tools to convert old <a href="http://en.wikipedia.org/wiki/ClarisWorks">ClarisWorks</a> and <a href="http://en.wikipedia.org/wiki/MacWrite">MacWrite II</a> documents to PDF for redaction.</p>
<p>It took some twiddling to come up with a conversion path that preserved good detail, but so far we&rsquo;ve gotten our best quality by using <a href="http://www.dataviz.com/">DataViz</a> <a href="http://www.dataviz.com/products/maclinkplus/">MacLinkPlus Deluxe</a> to convert all the files to Word for Windows format. That&rsquo;s enough to open them in <a href="http://www.rapidredact.com/">RapidRedact</a>. For testing <a href="http://www.appligent.com/redax">Redax</a>, though, we then use <a href="http://www.adobe.com/">Adobe</a> <a href="http://www.adobe.com/products/acrobat/">Acrobat</a> to convert those Word files to PDF. So far the conversion is pretty clean: Curly quotes and a variety of unusual characters inside the documents appear to convert cleanly, where in some other conversion paths they were lost or mistranslated. So far so good.</p>
<p>Next up: the redaction process itself.</p>
