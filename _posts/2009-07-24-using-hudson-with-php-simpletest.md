<p>ETD is now configured to be tested &amp; monitored in Hudson: http://waterhouse.library.emory.edu:8080/hudson/job/ETD/</p><p>It took a bit of work to get PHP SimpleTest results into a format that Hudson can process and give detailed, granular results like it does for the Django apps or anything else that natively produces jUnit-style testsuite xml, but now that I've figured it out for ETD it should be pretty easy to set up for any other existing PHP code-bases using simpletest for unit testing.</p><p>Here's what I'm using for the <strong>build</strong> command in the ETD Hudson configuration:</p><pre>cd trunk/tests/suites<br />rm -rf test-results<br />mkdir test-results<br />php xml_suite.php &gt; test-results/suite.xml<br />&nbsp;../simpletest/local/clean_simpletest_xml.php test-results/suite.xml <br />xsltproc ../simpletest/local/simpletest_to_junit.xsl test-results/suite.xml &gt; test-results/TEST-suite.xml</pre><p>Then, in the post-build options I enable <strong>Publish JUnit test result report</strong> with this path:</p><pre>trunk/tests/suites/test-results/TEST-*.xml<br /></pre><p>A few notes on the what/why/how.</p><p>Simpletest has an xml reporter that can be extended, but it's rather limited-- it outputs the xml as the tests run, so you don't have information on total pass/failure until the end.&nbsp; I extended the default XmlReporter to make one minor modification-- timing each test method as the test runs.&nbsp; To use this, include simpletest/xmltime.php and use XmlTimeReporter to run the report you wish to pass off to Hudson.&nbsp; For ETD, I created a new test suite script that includes all of the other unit test groups and uses the XmlTimeReporter.</p><p>I also discovered that if there are any errors, warnings, etc. before the test actually starts, those warnings are displayed before the xml output-- making the resulting file invalid xml.&nbsp; I wrote a very minimal php script to clean up anything before the xml prolog and put it into a system-err tag.&nbsp; Then I wrote an xslt to convert the xml generated by simpletest XmlReporter (with my test duration addition) into the testsuite xml format used by jUnit and understood by Hudson.&nbsp; I couldn't find any actual documentation of this format, just examples, so the output is an approximation and may miss some things, but Hudson is now correctly picking up failed tests and displaying test results for those failures, which I'm happy with (for now at least).</p><p>The additions to simpletest that make this conversion possible are now in our local simpletest subversion repository, so any project that is using simpletest should have access to them.</p><p>&nbsp;</p><hr width="100%" /><p>&nbsp;</p><p>Update January 29, 2010: for anyone who's interested but doesn't have access to our subversion, here's the content of the relevant two files.</p><p>xmltime.php:</p><pre><span class="tt">&lt;?php<br /><br />&nbsp; /**<br />&nbsp;&nbsp; * extend default XmlReporter to record &amp; report time to run each test method<br />&nbsp;&nbsp; */<br /><br />class XmlTimeReporter extends XmlReporter {<br />&nbsp; var $pre;<br /><br />&nbsp; function paintMethodStart($test_name) {<br />&nbsp;&nbsp;&nbsp; $this-&gt;pre = microtime();<br />&nbsp;&nbsp;&nbsp; parent::paintMethodStart($test_name);<br />&nbsp; }<br /><br />&nbsp; function paintMethodEnd($test_name) {<br />&nbsp;&nbsp;&nbsp; $post = microtime();<br />&nbsp;&nbsp;&nbsp; if ($this-&gt;pre != null) {<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; $duration = $post - $this-&gt;pre;<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; // how can post time be less than pre?&nbsp; assuming zero if this happens..<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if ($post &lt; $this-&gt;pre) $duration = 0;<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; print $this-&gt;_getIndent(1);<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; print "&lt;time&gt;$duration&lt;/time&gt;\n";<br />&nbsp;&nbsp;&nbsp; }<br />&nbsp;&nbsp;&nbsp; parent::paintMethodEnd($test_name);<br />&nbsp;&nbsp;&nbsp; $this-&gt;pre = null;<br />&nbsp; }<br /><br />}<br /><br />?&gt;</span><br /></pre><p>simpletest_to_junit.xsl:</p><pre>&lt;?xml version="1.0" encoding="utf-8"?&gt;<br />&lt;xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;
