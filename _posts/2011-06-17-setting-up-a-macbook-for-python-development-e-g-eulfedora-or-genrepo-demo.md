<p>Normally, I do my development on an Ubuntu workstation, which I've got pretty much set up and customized and I know how to manage it. &nbsp;But for my recent trip to the <a href="https://conferences.tdl.org/or/index.php/OR2011/OR2011main">Open Repositories 2011 conference</a>, I was given a loaner MacBook Pro as a travel machine (which, I have to admit, has been pretty nice to work on). &nbsp;This was particularly useful-- since I knew that at least a couple of the people who would be attending our pre-conference <a href="http://eulfedora.readthedocs.org/en/latest/tutorials/fedora.html">tutorial</a> on programming with <a href="https://github.com/emory-libraries/eulfedora">eulfedora</a> were planning to do their development on OSX, this gave me a chance to test installing everything ahead of time.</p><p>What follows are the my setup notes for getting a new MacBook Pro (64 bit, Snow Leopard) set up and configured so I could do python/eulfedora development work. &nbsp;I'm sure my notes are idiosyncratic, but I'm posting them in case I ever need to do this again, and in case they are helpful to others (since it was invaluable to me to find the odd blog post or forum discussion addressing some of the various snags &nbsp;and issues I ran into).</p><p>First, I installed Google Chrome (I'm just not used to Safari), and then Python 2.7 using the Mac installer from the <a href="http://www.python.org/download/releases/2.7.2/">Python downloads</a>. &nbsp;(I think now that a new enough version of Python was probably already installed and I could have just used that, but I'm not sure how to easily confirm.)</p><p>Since I wanted to work with eulfedora and genrepo-demo (among other things), which are both hosted on the <a href="https://github.com/emory-libraries">emory-libraries github</a>, I set up git as described here: <a href="http://help.github.com/mac-set-up-git/">http://help.github.com/mac-set-up-git/</a></p><p class="p1">Since eulfedora and the other pytho-eul* libraries are configured to be easily set up with virtualenv and pip, I grabbed those tools first with easy_install:</p>
<p class="p2"><span style="white-space: pre;">	</span>sudo easy_install virtualenv pip</p>
<p class="p1">One of the python libraries we use is <a href="http://lxml.de/">lxml</a>, which is sometimes a pain to install, since it needs to be compiled against libxml and libxslt. &nbsp;I installed the OSX developer/xcode tools from the application install CD so that I could pip install packages that need to be compiled.</p>
<p class="p1">I also discovered (after some trial and error) that you have to tell Python/setup tools&nbsp;not to try to compile things for ppc, since current versions of xcode don't actually support it - this can be done by setting ARCHFLAGS to not include ppc:</p>
<p class="p3"><span style="white-space: pre;">	</span>ARCHFLAGS="-arch i386 -arch x86_64"</p>
<p class="p4">I think that libxml and libxslt probably were already installed on the system, but if you are on a system that doesn't have them, you can use STATIC_DEPS as a work-around. &nbsp;The example I saw suggested you run this step first, without sudo:</p><p class="p4"><span style="white-space: pre;">	</span>STATIC_DEPS=true easy_install lxml</p><p class="p4">This will download the libraries you need available to install lxml. &nbsp;After that you should be able to install lxml normally, via sudo easy_install or pip.</p><p class="p4">The current version of <a href="https://github.com/emory-libraries/genrepo-demo">genrepo-demo</a> does a little bit of mimetype checking on uploaded files before ingesting them into the repository. &nbsp;This uses python-magic, which needs information about file types. &nbsp;Do get that working, I installed darwinports from&nbsp;<a href="http://www.macports.org/">http://www.macports.org/</a>&nbsp;and then all I had to do was:</p>
<p class="p3"><span style="white-space: pre;">	</span>sudo port install file</p>
<p class="p4">I think that was basically it for the libraries and configuration side of things.</p><p class="p4">I primarily use Emacs as my editor, and I took a look at Aquamacs, since I know other Mac users who like it - but it seemed foreign and not quite what I wanted, so I was quite pleased to discover that GNU Emacs was available for OSX - and I was able to copy over all my emacs configurations and customizations (including python and django modes, and autocomplete) and basically have things work.</p><p class="p4">I also spent a little bit of time playing with <a href="https://github.com/emory-libraries/fedorafs">fedorafs</a> - I knew there was some implementation of FUSE for OSX, but not much more than that. &nbsp;The downloaded installer for <a href="http://code.google.com/p/macfuse/">macfuse</a> should work, and I think it should also be installable via darwinports:</p><p class="p4"><span style="white-space: pre;">	</span>sudo port install macfuse</p><p>However, apparently macfuse on 64-bit Snow Leopard is still a bit experimental, so I had to find an alternate built version elsewhere. &nbsp;Of course, then I discovered that the Python FUSE bindings I had been using don't work for macfuse (and haven't been updated in a while either), so I am looking at updating fedorafs to use <a href="http://code.google.com/p/fusepy/">fusepy</a> - but that is a subject for a different post.</p>
