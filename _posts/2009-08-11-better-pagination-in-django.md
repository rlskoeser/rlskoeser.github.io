<p>The pagination features provided with django are adequate bit a bit invasive in my opinion.&nbsp; They require to to modify views to handle paging in most cases and this strays out of DRY methodologies.&nbsp; Indeed most actual projects seem to use Pagination middleware rather than the pagination services with django direction.</p><p>One of the most commonly used pagination middlewares in django is <a href="http://code.google.com/p/django-pagination/" target="_blank">Django-Pagination</a> which provides simple Digg Style pagination handled entirely at the template level.&nbsp; As such it can be included in most projects without having to modify views at all.</p><p>The biggest downside I can see is that it seems to pass page params as a query string and if you don't like that design in your app it could be a roadblock, if you don't like it however there are other apps out there that will handle paging via clean URLs.&nbsp; It could be that this code handles it as well but I haven't looked very hard into it as I don't the querystring format for paging.</p><p>Additionally you can find a lot of style libraries out there for displaying your paging elements on your site.&nbsp; A very useful site I found is <a href="http://mis-algoritmos.com/2007/03/16/some-styles-for-your-pagination/" target="_blank">Mis Algoritmos</a>, a spanish language site but easily usable for our purposes.&nbsp; It has links to a dozen or so css styles that will work with this paging module.</p>
