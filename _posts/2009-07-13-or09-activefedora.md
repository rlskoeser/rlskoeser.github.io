<h3>When Ruby met Fedora: ActiveFedora</h3>
<p>Matt Zumwalt. MediaShelf<br />view presentation materials: <a href="http://hdl.handle.net/1853/28502"><strong><code>http://hdl.handle.net/1853/28502</code></strong></a><a href="http://hdl.handle.net/1853/28421"></a></p><p>Matt Zumwalt's vision is for long-lived content with specialized views and interfaces for specific users-- treat the software as perishable but the data as durable.&nbsp; The goal is to allow us to focus on the user experience and not worry about infrastructure like Fedora.&nbsp; In fact, earlier in the Open Repositories conference Matt had a separate presentation on this idea: <a href="http://hdl.handle.net/1853/28418">Many Lightweight Views into Complex Repository Content: Enabling Rapid Application Development for Fedora Repositories</a>.</p><p>ActiveFedora is not just a ruby wrappper for the Fedora APIs, but the equivalent of IRM for SQL dbs.</p><p>Why choose ruby instead of python?</p><ul><li>flexible, dynamic - introspection, add methods (relatively easy in python, but structures in ruby avaliable)</li><li>jruby - lets you use java libraryes; more stable and "cooler" than jython</li><li>rails integration with prototype, scripaculous, and now jquery</li></ul><p>MediaShelf has built everything around real applications and projects.&nbsp; As they need new features for their clients, they build on to ActiveFedora.&nbsp; Currently ActiveFedora supports qualified Dublin Core (built-in right now, really should be a separate gem).&nbsp; They haven't yet done any work with MODS, but there are java libraries for MODS theat they can pull in through jruby.&nbsp; For agile development, ActiveFedora makes it easy to start Fedora and then throw things away (dev/test systems, throwaway objects); comes with a pre-fab Fedora with solr &amp; demo objects.&nbsp; ActiveFedora includs a solr service; serialize to solr functionality is handled by datastream models.</p><p>The idea is to hide complexity and make ActiveFedora look familiar to Ruby on Rails so that every developer doesn't have to be a Fedora expert.</p><p><a href="http://mediashelf.us/activefedora/">http://mediashelf.us/activefedora/</a></p><p>&nbsp;</p><hr /><p>Relevance/applicability for us?&nbsp; ActiveFedora is not directly applicable, since we are moving away from Rails and doing new development in Django; but MediaShelf's ActiveFedora may serve as a useful model or reference point as we will hopefully be working to create something similar in python.&nbsp; Also, there is considerable interest in this project in the Repositories community, and may be other groups and institutions working on a python-based project similar to this, so there are some possibilities for collaboration and creating a more general-purpose interface to Fedora to further this same goal of simplifying the process of creating quick, light-weight interfaces to Fedora content.&nbsp; </p><p>&nbsp;</p>
