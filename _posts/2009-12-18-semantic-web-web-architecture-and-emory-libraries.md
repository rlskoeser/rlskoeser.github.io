<p>I’ve been itching for a little while to start using <a href="http://en.wikipedia.org/wiki/Resource_Description_Framework">RDF</a> and related <a href="http://en.wikipedia.org/wiki/Semantic_Web">Semantic Web</a> technologies here at <a href="http://web.library.emory.edu/">Emory Libraries</a>. It seems like an excellent fit for organizing, linking, and sharing some of the data that libraries are wont to collect. But alas, where to start; where to start?</p>

<p>See, as might be expected, there’s a ton of data here at the library. There’s not a bunch of work right now in my department for RDFizing it, though. And to a degree that’s sensible: We have a lot of other commitments, and there are a lot of sensible things for us to be working on.</p>

<p>At the same time, more and more information is being added to the Semantic Web. Tons of folks are getting linked up with <a href="http://www.foaf-project.org/">FOAF</a> profiles, and between <a href="http://projects.apache.org/">apache</a>, <a href="http://pypi.python.org/">PyPI</a>, and <a href="http://git.gnome.org/cgit/">gnome</a>, more and more projects are getting <a href="http://trac.usefulinc.com/doap">DOAP</a> data. I think the final straw for me, though, was when the <a href="http://www.loc.gov/">Library of Congress</a> started providing <a href="http://www.w3.org/2004/02/skos/">SKOS</a> representations of their subject headings. This is no longer just a personal interest of mine: It’s something we really need to be involved in if we want to remain a cutting-edge university library.</p>

<p>So anyway, I was particularly thinking of it a week or two ago when some folks started up a local <a href="http://www.meetup.com/The-Atlanta-Semantic-Web-Meetup/">Semantic Web Interest Group</a>. I attended along with two other people from Emory University Library staff. I suggested to the people there — mostly relative newcomers to the Semantic Web — that setting up a FOAF file is a good way to get started and get yourself linked up. And while I’ve had a personal (mostly-unlinked) FOAF file at home, it reminded me yet again that I probably ought to have a work one, too. One I can use to sow seeds of RDF that I can start to spread to my work projects, perhaps getting people using the technologies. One that I can use to start people thinking about how we can benefit from integrating the principles into our other development efforts.</p>

<p>So that led us to today. Today was an <a href="http://www.extremeprogramming.org/rules/iterationplanning.html/">iteration planning meeting</a> at Emory Libraries Software, and the last chunk of iteration planning day here is left flexible for Other Stuff™, mostly high-level planning about upcoming work and organizing projects and ideas. I like to spend it <a href="https://techknowhow.library.emory.edu/preplanning-systems-driven-projects/preplanning-systems-driven-projects">brainstorming</a> and whatnot. Today, though, I decided to start looking for a way to get my own web space where I can start seeding a local RDF graph. Nothing production-scale, perhaps, but just something to start exposing people to the technologies.</p> 

<p>My quest led me to a manageable web home for myself here <a href="http://wilson.library.emory.edu/~branker/">on a development server</a>, but I didn’t really have the time and scope to organize anything more grand-scale for the library. What I did, though, is get in a couple of interesting conversations.</p>

<p>In particular I chatted with Stacey Martin, who’s been working as the Project Strategist for the Libraries’ web site. There’s a website redesign in progress, to be released Real Soon Here, and it gets us a lot closer to <a href="http://www.w3.org/Provider/Style/URI">Cool URIs</a> for large chunks of our web face. Me, though, I want to take it further. While Stacey’s done a great job organizing the user experience of our website, I want to see more work done toward organizing the URIs — the <em>technology’s</em> interface to our website. I want to see <a href="http://www.w3.org/standards/webarch/">Web architecture</a>.</p>

<p>So I babbled with Stacey a bit about my ideas. She seemed pretty receptive. It’s a big chunk of work: We have a huge web presence here at Emory Libraries, spread across hundreds of diverse websites. And while there’s been some good human interface design, there really hasn’t been much thought put into Web architecture and technology interface design, pretty much at all.</p>

<p>I’ve been reading up on web architecture for years. Perhaps not following it as closely as people actively engaged in creating it, but that’s mostly been for a lack of an appropriate position to make things happen. Here and now I think I might just have a chance to make a difference.</p>

<p>I’ll talk more about my specific ideas and the current state of affairs in future posts. For now, though, I’ll just say I think it’s an exciting task, and while I don’t expect much to change overnight, I’m really excited to be talking about pushing things in the right direction.</p>
