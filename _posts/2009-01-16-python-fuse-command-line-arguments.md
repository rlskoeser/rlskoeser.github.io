<p>It took me a while to figure how to add custom arguments in python-fuse, so I want to document how I got it to work.  Maybe it would have been compeltely clear to someone who is not so new to python, but the examples didn't seem complete or obvious to me.</p> <p>An important thing that helped me, buried in the <a href="http://fuse4bsd.creo.hu/fuse-python-0.2-doc/">python-fuse api docs</a>, is the "basic usage" statement for the Fuse class:</p><ul><li>instantiate</li><li>add options to parser</li><li>call parse</li><li>call main</li></ul><p>Here's an example of the code that adds a new option:</p> <pre>server.parser.add_option(mountopt="host", metavar="HOSTNAME", default=server.host,
  help="fedora server host name [default: %default]")</pre> <p>(<em>Note:</em> This is in the main() function in the FedoraFs.py file; and server is an instance of the FedoraFS class.  This follows the structure of all the examples and tutorials I saw.)</p><p>Now when I run <strong>./FedoraFs.py -h</strong> I get an additional option:</p> <pre>    -o host=HOSTNAME       fedora server host name [default: localhost]</pre> <p>The thing I couldn't figure out from the documentation and examples was how to actually use the values passed in on the command line.  Here's how I got it to work (there may be other or better ways):</p> <ol> <li>Add the new options as variables to my fuse class and set the defaults in my init function, e.g.   <pre>    self.host = "localhost"</pre> </li> <li>Instruct parse to set option values in the current instance of FedoraFS fuse class:  <pre>server.parse(values=server, errex=1)</pre></li> <li>Use options variables in fuse class main function, <em>after</em> they have been set (rather than in the init).  In this case, it looks like this:  <pre>self.fedora = client.getClient("http://" + self.host + ":" + self.port + "/fedora",
                               self.username, self.password, self.version)</pre></li> </ol><p>Note that in the initial code sample where I am adding the option, I'm using the server variable server.host (unlike all the examples, which seem to use plaintext defaults here), so that I only have to define my default values once.  Perhaps it would also work to define the server variables in the init but leave them blank and let the parser set all the values (it's not clear to me if the parser sets all the values, including defaults, or only the non-defaults that are set by the user).</p>
