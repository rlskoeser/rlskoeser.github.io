<p>The process for generating OAI-MARC for primo requires 2 jobs.  Loading the data to primo is a 3rd step</p><p>1. prepForPrimo lib/tasks/prepForPrimo.rake</p><p>This job will process any items in the "Ready For Repository" state and generate marc.xml files in the primo/current directory</p><p>This job should be run nightly after the move_to_lsdi.rake</p><p>Optional parameter item_list=&lt;comma seperated list of item id&gt; -- this will generate marc xml files for all listed items regardless of the current item state</p><p>example  DigWF:prepForPrimo item_list=1377,1525</p><p> </p><p>2. batchForPrimo lib/tasks/batch_primo.rake</p><p>This job scans the primo/current directory for marc.xml files and will generate a single oai-marc file for ingest into Primo</p><p>The oai-marc file is then tar gzipped and placed in the primo/pickup directory, the primo/current directory is then renamed as date(Y-m-d) </p><p>This job should be run periodically, weekly or monthly should be sufficient to load new data to primo</p><p> </p><p>3.  loading data to primo</p><p>Bernardo is grabbing the emory_kirtas_&lt;timestamp&gt;.tar.gz loading it into primo and deleting the file</p><p>To load data to production primo it will be necessary to recreate this file.  Simply tar gzip the lsdi_primo_&lt;timestamp&gt;.xml files found in each previous batch directory</p><p>To recreate a recreate the lsdi_primo_&lt;timestamp&gt;.xml file you will need to run the batch_primo.rake task against the desired batch directory.  The is not a feature of the current script and will require a code change.</p><p> </p>
