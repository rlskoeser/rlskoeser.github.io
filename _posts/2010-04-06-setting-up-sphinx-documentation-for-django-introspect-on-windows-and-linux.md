<p><a href="http://sphinx.pocoo.org/" target="_blank">Sphinx</a> is, hands down, one of the best tools I've run across for generating project documentation out there but I stumbled a bit at first trying to get it setup to introspect my Django applications on my Windows development box.&nbsp; After a bit of toying I thought I'd share what I learned and how to get it to work.</p><p><strong>Sphinx Make Files</strong></p><p>For convienence Sphinx constructs make files as part of `sphinx-quickstart` and if you're executing the command on a windows machine it detects it and asks if you'd like to make a Make.bat file in addition to the normal Make file.&nbsp; This operations much like the normal Make file and the concepts of what needs to be done to both to enable introspection are the same.&nbsp; Essentially you have to add your Django application and any exteranals to the Python Path in order for them to work.</p><p>So lets say your Sphinx documentation root directory is like so:</p><ul><li>source/doc&nbsp; - Sphinx documenation root</li><li>source/foo - Djano application</li><li>source/foo-externals&nbsp; - Externals and such.</li></ul><p>To add your Django app to your python path you need to add the following line in source/doc/Make.bat</p><pre class="tt">set PYTHONPATH=%PYTHONPATH%;..;../foo-externals</pre><p>This is just a normal windows set command like you would with any other PATH variable.&nbsp; If you are running `sphinx-build` from command line you can just type that in before running it and your app and externals will be on the path.</p><p>Something similar needs to be done in the source/doc/Make as well but the Unix equivalent is:</p><pre class="tt">export PYTHONPATH=..:../foo-externals</pre><p>Additionally for introspection of Django apps, Django itself will need to load something into DJANGO_MODULE_SETTINGS but this can be a blank file.&nbsp; So create a blank py file called something like source/doc/djangosettings.py </p><p>Then add the following line to source/doc/Make.bat</p><pre class="tt">set DJANGO_SETTINGS_MODULE=djangosettings</pre><p>and in source/doc/Make add</p><pre class="tt">export DJANGO_SETTINGS_MODULE=djangosettings</pre><p>That is pretty much all you need to do.&nbsp; Sphinx should be able to introspect anything you setup to be read correctly, you can see the Sphinx docs for exactly how to do that.</p><p>&nbsp;</p>
