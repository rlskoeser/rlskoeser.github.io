discovered the solution in the documentation and found the culprit was the default manager.  For sensible reasons a generic manager is returned on related sets, primarily because it usually needs to get everything related to an object and not just whatever a custom manager filters on.  If you need your Custom Manager methods in your related set all you need to do is set "<span class="tt">use_for_related_fields = True" on the custom manager and you'll get that from now on with a related set.<br /></span></p>
