<div dir="ltr"><font face="Tahoma" size="2" color="#000000"><font face="tahoma">I</font>f 
we had the OPUS data in a CMS like Drupal, this could be a useful framework for building better links to library resources.  For example, perhaps someone could write something like a Greasemonkey script to 
automatically check course numbers against the LibGuide site.  Then users could 
click on the course numbers to open the corresponding Libguide if there is one.  
</font></div>
<div dir="ltr"><font face="Tahoma" size="2" color="#000000"></font> </div>
<div dir="ltr"><font face="Tahoma" size="2" color="#000000">If there isn't one, perhaps the developer could write a script to generate a generic course guide <font face="Tahoma" size="2" color="#000000">automatically and/or </font>automatically 
generate an email request to Collection Mgt for a handcrafted one.  A generic course 
libguide would include info standard to all disciplines or that specific 
discipline, e.g., reference works, and the content specific to the course could 
be generated based on the title, course description, and texts mentioned.  
</font></div>
<div dir="ltr"><font face="Tahoma" size="2" color="#000000"></font> </div>
<div dir="ltr"><font face="Tahoma" size="2" color="#000000">That might mean that the 
libguides get further modularized with additional metadata or taxonomic tags 
added to each guide, module or node (which could be a guide, a tab in a guide, 
or a section within a tab).   </font></div>
<div dir="ltr"><font face="tahoma" size="2"></font> </div>
<div dir="ltr"><font face="tahoma" size="2">It should be possible as well to have any 
automatically generated course guide analyzed for richness of data results to 
see whether manual attention is necessary to fill in a gap via a libguide 
module.  </font></div>
<div dir="ltr"><font face="tahoma" size="2"></font> </div>
<div dir="ltr"><font face="tahoma" size="2">In addition, of course, we could enable 
comment, request, and feedback forms.  </font></div>
<div dir="ltr"><font face="tahoma" size="2"></font> </div>
<div dir="ltr"><font face="tahoma" size="2">And, it would be possible to track clicks 
to see what part of the structure or specific course guides gets used most 
often, which info gets forwarded and to whom and automatically generate 
appropriate charts and graphs so that humans could analyse the user data.  
</font></div>
