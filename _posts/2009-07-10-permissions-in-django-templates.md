<p>I struggled with user permissions in Django templates and found an interesting "gotcha" that I thought might be worth passing on in case someone were to make the same incorrect assumptions from the documentation that I did.</p><p>First, for reference the <a href="http://docs.djangoproject.com/en/1.0/topics/auth/#authentication-data-in-templates" target="_blank">Django Template authentication</a> documentation is needed.</p><p>Where I went wrong is in understaning what portion of a permission object you use in a template.</p><p>Models recieve custom permissions in the Meta subclass like so:</p><p>class Meta:<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; permissions = (<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ("can_update_plan", "Can update planning info"),<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ("can__update_development", "Can update development info")<br />&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; )</p><p>and my confusion came in which part of the Tuple gets used in the template.&nbsp; In the admin interface you ONLY see the second ([1]) tuple item.&nbsp; So in the case of the permission  <em>("can_update_plan", "Can update planning info") </em>you would only see "Can update planning info" in the admin permissions interface.</p><p>What's important to know is you have to use the first part of the tuple in the template NOT the display section.&nbsp; This is easy enough to find out in the case of custom permissions but for provided permissions you'll need to look in the source DB to get the short version.</p>
